Docker Compose setup:

The repo lives on GitHub and gets pulled to the NAS manually
The NAS runs docker compose build && docker compose up -d to deploy
There is a .env file on the NAS (not in the repo) that holds secrets like DB_PASSWORD, API keys, etc.
The compose file uses ${VARIABLE:-default} syntax so it works without the .env if defaults are fine for dev
Database:

PostgreSQL running as a Docker service named db
Credentials via env vars: POSTGRES_DB, POSTGRES_USER, POSTGRES_PASSWORD
A docker/init.sql file in the repo initialises the schema on first run
A named volume (pgdata) persists the data across restarts
Healthcheck: pg_isready -U <user>
Services needed:

db — postgres:16-alpine
api — Node.js Express backend, built from a Dockerfile, depends on db
web — React/Vite frontend, built from a Dockerfile using nginx to serve the static build

Multi-stage builds (build stage + slim runtime stage)
API: builds with pnpm, runs the compiled JS with node
Web: builds with pnpm/vite, copies dist/ into an nginx image

CACHE_BUST=$(date +%s) docker compose build api web && docker compose up -d
